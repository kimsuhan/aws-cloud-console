## Design Context

### Users
This product is for individual developers who need to access and manage multiple AWS terminal workflows from one desktop app. The core job is to connect to several AWS environments, open and manage shell and tunnel sessions, and maintain a clear view of current operational state without losing context.

### Brand Personality
This app does not need consumer-style branding. Treat it as a pragmatic utility product with a direct, technical voice. The target emotional tone is trust and control, so the interface should feel dependable, clear, and operational rather than playful or decorative.

### Aesthetic Direction
Preserve the existing AWS-green-led palette and the current high-density developer-tool character. Support both dark mode and light mode, but keep the visual system technical and grounded in AWS console sensibilities rather than soft or lifestyle-oriented UI. Avoid pastel, pink, or delicate color directions. Favor structured panels, explicit state markers, and strong contextual cues that help users understand which profile, region, session, or tunnel they are operating on.

### Design Principles
1. Prioritize operational clarity over visual flourish. Active profile, region, session state, and available actions should always be easy to scan.
2. Design for multi-session control. The interface should help users manage several AWS shells and tunnels without losing orientation.
3. Reinforce trust through explicit state. Show selections, statuses, and consequences clearly so the app feels controlled and dependable.
4. Keep the aesthetic technical and AWS-adjacent. Use AWS-green accents intentionally, avoid soft palettes, and maintain a serious developer-tool tone.
5. Build adaptable ergonomics into the UI. Support both dark and light themes, and provide user-adjustable interface scale options such as 100%, 110%, and similar density controls.
